GET

Код ответа:

200 OK — успешное получение данных.
404 Not Found — ресурс не найден.
400 Bad Request — некорректный запрос.


Заголовки (Headers):

Content-Type (например, application/json).
Cache-Control, ETag (для кэширования).
Content-Length (размер тела ответа).


Тело ответа (Body):

Данные ресурса (например, JSON с объектом или списком).
Пустое, если ничего не найдено (зависит от API).



POST

Код ответа:

201 Created — ресурс успешно создан.
400 Bad Request — ошибка в данных запроса.
409 Conflict — конфликт (например, ресурс уже существует).


Заголовки (Headers):

Content-Type (тип возвращаемых данных).
Location (URL созданного ресурса, часто при 201).


Тело ответа (Body):

Данные созданного ресурса (например, JSON с ID и полями).
Может быть пустым (зависит от API).



PUT

Код ответа:

200 OK — ресурс обновлен.
201 Created — ресурс создан (если ранее не существовал).
404 Not Found — ресурс не найден.


Заголовки (Headers):

Content-Type (тип данных ответа).
Location (иногда, если ресурс создан).


Тело ответа (Body):

Обновленный ресурс (JSON с новыми данными).
Может быть пустым (например, при 204 No Content).



PATCH

Код ответа:

200 OK — ресурс частично обновлен.
404 Not Found — ресурс не найден.
400 Bad Request — некорректные данные.


Заголовки (Headers):

Content-Type (тип возвращаемых данных).


Тело ответа (Body):

Обновленный ресурс или только измененные поля (JSON).
Может быть пустым (например, при 204 No Content).



DELETE

Код ответа:

200 OK — ресурс удален.
204 No Content — удаление успешно, без возврата данных.
404 Not Found — ресурс не найден.


Заголовки (Headers):

Content-Type (если есть тело ответа).


Тело ответа (Body):

Обычно пустое (особенно при 204).
Иногда JSON с подтверждением удаления (зависит от API).
